Model & Project Workflow

How the Student Exam Score Prediction Model was Designed and Built

1. Problem Definition

Understanding the impact of academic, personal, and socioeconomic factors on student exam performance and building a model to predict scores with high accuracy.

2. Dataset Preparation

  • Loaded data from CSV file containing features like Hours Studied, Attendance, Sleep Hours, etc.
  • Handled missing values using forward fill and mode-based imputation.
  • Categorized columns into numeric, categorical, and ordinal types.

3. Feature Engineering & Encoding

  • Applied Ordinal Encoding for ranked features (e.g., Parental Involvement).
  • Used OneHot Encoding for nominal categories (e.g., Gender, Internet Access).
  • Standardized numerical columns using StandardScaler.

4. Model Training

  • Split data into training and testing sets (80/20).
  • Trained a Gradient Boosting Regressor inside a pipeline.
  • Evaluated with R¬≤ Score, MAE, and RMSE metrics.

5. Model Deployment

  • Used joblib to save the trained pipeline.
  • Created Django views to allow training and prediction via UI.
  • Bulk prediction supports CSV uploads and shows predictions in-browser with download option.

6. Future Enhancements

  • Support for XGBoost or RandomForest alternatives.
  • Visual dashboards with SHAP values for interpretability.
  • Admin dashboard for model retraining and dataset uploads.
